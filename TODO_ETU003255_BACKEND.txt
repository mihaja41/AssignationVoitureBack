3) ✅ FAIT - Logique d'attribution véhicule (Service)
==============================

Fichier créé :
  - service/PlanningService.java

Algorithme implémenté dans genererPlanning(date) :
  Pour chaque réservation NON_ASSIGNE à la date donnée :
    1. Chercher véhicules où nb_places >= reservation.passengerNbr
    2. Vérifier DISPONIBILITÉ :
       → Exclure véhicules déjà ASSIGNE dont heure_retour > heure_depart de cette réservation
       → (évite double assignation / conflit planning)
    3. Si >= 2 véhicules éligibles → priorité DIESEL (TypeCarburant.D)
    4. Si >= 2 Diesel éligibles   → choix aléatoire (random) parmi eux
    5. Si 1 seul Diesel           → on le prend
    6. Si 0 Diesel                → random parmi les autres disponibles
    7. Si 0 véhicule disponible   → réservation reste NON_ASSIGNE

Fichier modifié :
  - repository/VehiculeRepository.java
      → Ajout findAvailableVehicules(int passengerNbr, LocalDateTime heureDepart)
        Requête SQL : véhicules avec nb_place >= N et sans conflit horaire

Fichier modifié :
  - repository/ReservationRepository.java
      → Ajout findAssignedByDate(date)      : réservations ASSIGNE pour la date
      → Ajout findUnassignedByDate(date)    : réservations NON_ASSIGNE pour la date

POINT IMPORTANT - Hotel vs Lieu
==============================

POURQUOI on garde Hotel séparément :
  - La table "hotel" et la classe Hotel.java sont utilisées par l'autre repo frontend
  - Reservation.hotel_id = d'où vient le client (son hôtel)
